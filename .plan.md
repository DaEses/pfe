# HR Interview Chatbot with Emotion Detection - Implementation Plan

## Overview
Create a Jupyter notebook (`chatbot/hr_interview.ipynb`) that runs an HR interview chatbot powered by Ollama, integrated with the existing webcam emotion detection model. The chatbot interviews candidates for a user-specified job role and adapts its behavior based on real-time facial emotion analysis.

## Architecture

```
┌─────────────────────────────────────────────┐
│            Jupyter Notebook                  │
│                                              │
│  Cell 1: Imports & Config                    │
│  Cell 2: Emotion Detection (reuse model)     │
│  Cell 3: Ollama Chat Functions               │
│  Cell 4: Interview Loop (text + webcam)      │
└──────┬──────────────┬───────────────────────┘
       │              │
       ▼              ▼
  Ollama API     Webcam + CNN
  localhost:11434  binary_emotion_model.h5
```

## Notebook Cells Breakdown

### Cell 1 — Imports & Configuration
- Import: `cv2`, `numpy`, `keras`, `requests`, `json`, `threading`
- Config: Ollama URL, model name, webcam index, emotion threshold

### Cell 2 — Emotion Detection Module
- Load `binary_emotion_model.h5` from `../emotiondetection/models/`
- Load Haar cascade for face detection
- Background thread: captures webcam frames, runs emotion prediction continuously
- Stores the latest emotion state (irritated/neutral + confidence) in a shared variable
- Functions: `start_emotion_monitor()`, `stop_emotion_monitor()`, `get_current_emotion()`

### Cell 3 — Ollama Chat Functions
- System prompt: HR manager persona with instructions to:
  - Ask role-specific interview questions one at a time
  - Evaluate answers professionally
  - Adapt tone if candidate appears stressed/irritated (from emotion data)
  - Provide a final evaluation summary at the end
- `chat_with_ollama(messages)` — sends conversation history to `POST /api/chat`
- `build_system_prompt(job_role)` — generates the system prompt with the specified role

### Cell 4 — Interview Loop
- Prompt user to enter the **job role** they're applying for
- Start webcam emotion monitor in background thread
- Display webcam feed in a separate OpenCV window (reuse existing pattern)
- Text-based chat loop in the notebook:
  1. HR bot asks a question
  2. User types response via `input()`
  3. Current emotion state is appended as context to the message sent to Ollama
     (e.g., "[Emotion: IRRITATED, confidence 78%]" added to system context)
  4. Ollama generates next question/response
  5. Repeat for ~5-8 questions
  6. Bot provides final evaluation summary
- Stop webcam on exit

## Key Design Decisions

1. **Webcam runs in a background thread** — keeps the notebook responsive for text input
2. **Emotion is passed as context** to Ollama, not shown to the candidate — the bot silently adapts
3. **Uses `requests`** to call Ollama API directly (no extra dependencies like `langchain`)
4. **Conversation history** is maintained as a list of messages for multi-turn chat
5. **Reuses the existing trained model** — no retraining needed
